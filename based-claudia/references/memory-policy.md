# Memory Policy

Durable learning is proposal-driven. Do not silently promote memory from a conversation.

## Draft Requirements

- Scope: session, feature, repo, user, or global.
- Claim.
- Provenance evidence.
- Confidence.
- Supersession rules.
- Retirement conditions.
- Privacy notes.
- Allowed readers.

## Promotion Rules

Promote only after explicit approval, validation where possible, privacy review, and supersession review. Do not store secrets, credentials, private keys, regulated data, or unnecessary private user state.

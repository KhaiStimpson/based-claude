- `../../references/memory-policy.md`

## Workflow

1. Identify the source: explicit user statement, validated trace, repository evidence, or external reference.
2. Draft a memory or improvement proposal with scope, provenance, confidence, supersession, retirement conditions, and privacy notes.
3. Use `based-claudia memory draft --title "<title>" --summary "<claim>"` for durable drafts.
4. Do not promote durable memory without explicit approval.
5. Prefer small reversible improvements over broad prompt rewrites.

Do not store secrets, credentials, private keys, regulated data, or unnecessary private user state.

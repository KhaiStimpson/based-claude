# Self-Improvement Protocol

Self-improvement changes skills, scripts, policies, memory, adapters, evaluators, or workflow rules only from validated trace evidence.

## Pipeline

1. Trace: capture objective, evidence, validation, risk, and outcome.
2. Proposal: define target surface, before evidence, expected benefit, validation plan, risk, rollback, and retirement condition.
3. Validation: run deterministic checks that can falsify the change.
4. Review: require independent review for evaluator, safety, memory-policy, executable script, hook, MCP, or trust-boundary changes.
5. Promotion: apply only with approval when the validation and review record support it.
6. Retirement: remove or supersede stale improvements.

## Rejection Modes

- Evaluator gaming.
- Prompt or policy drift.
- Memory leakage.
- Automation without kill criteria.
- Improvement based on a single ambiguous trace.
- Hidden dependency on a local environment.

# Action-State Handoff Template

Use this when context moves between planning, implementation, validation, review, memory promotion, or a long-running continuation.

```md
## Action-State Handoff
- Objective:
- Current owner / mode:
- Instruction contracts:
- Authority / approval boundary:
- Scope:
- Exclusions:
- Current state:
- Files read:
- Files changed:
- Decisions:
- Evidence:
- Validation:
- Risks:
- Privacy / safety notes:
- Tool-trust notes:
- State invariants:
- Next action:
```

## Rules

- State what changed, not every sentence said.
- Include exact paths, commands, outputs, and uncertainty when they matter.
- Mark inferred facts as inferred.
- Preserve blockers, replanning triggers, and skipped checks.
- Keep it short enough for the next worker to act immediately.
